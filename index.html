The Basics of the Web and HTML

Back to Top

WWW - World Wide Web

The world wide web is a collection of HTML documents. At the time of this lesson approximately 30 billion documents make up the web. HTML, or hypertext markup language, is the language of the world wide web. It is the basic building blocks for the documents on the web. When you are browsing the web and you enter a URL into your browser your computer sends an HTTP request to server corresponding to that address. The server responds back by sending the file or webpage you requested with the URL.

HTML

Hypertext markup language is the heart of the web. It is made up of text content (what you see on a page), markup (what tells you browser how to arrange the content on the page), references to other documents (images and videos), and links to other pages.

HTML Markup

Markup is the tags and attributes that tell the browser how to display a webpage. It has the following format:

<tag>Contents</tag>

Other examples of tags are:

  • Strong (Bold): <strong>
  • Emphasis: <em>
  • Line Break: <br>

Computers are Stupid

A computer can only interpret instructions exactly as you have entered them. It does not infer, and it does not guess.

HTML Attributes

Attributes provide more information about an HTML element. They have the following format:

<tag attr="value">Contents</tag>

Examples include:

  • Anchor tag: <a href="URL">Link Text</a>
  • Image tag: <img src="URL" alt="TEXT"> (Note that the image tag is a void tag. It does not have contents and does not require a closing tag.)

Whitespace - Inline vs. Block

Inline elements are exactly like they sound. They work inline without breaks. Block elements create an invisible box around its contents and break to a new line.

Holding it All Together

HTML is like the structure of a home. It defines the where the walls are and what each room is. It is the langauge and syntax that builds the structure and content of a page.

CSS is the style of the home. It determines the wall color and what is decorating the home. Is defines how elements on a page look. Such as the text color, fonts, sizes, and backgrounds.

However, not all HTML elements are visible on a page. Some are hidden such as the <head> section.

Boxes, Boxes, Boxes!

All HTML elements are contained within an invisible box. These boxes can be nested deeply within each other. When viewing structured HTML, the more indented the line is in the code, the more deeply it's "box" is nested.

Browser Developer Tools

The tree like structure of a webpage's HTML is easily viewed within Chrome's Dev Tools.

If you dig deep enough into the code within a browser's developer tools, you can always find exactly what is displayed on the page.

The DOM or document object model, is the tree-like code that you see displayed within a browser's developer tools. It is how the browser interpreted the HTML and CSS to construct the page.

CSS

CSS stands for "Cascading Style Sheets". The "cascading" is what is important to remember.

Don't Repeat Yourself

Avoiding repetition allows code to be more concise and consistent and will also save a great deal of time while writing the code. If many elements on a webpage are to have the same style, then they can all be assigned the same "class" so that they can all inherit the same style from the CSS stylesheet. Also, if changes are made later the changes will apply universally across the HTML rather than having to edit each individual element.

Preventing errors is also a great reason to avoid repetition in code. While writing code, if the same lines are repeated over and over again to achieve the same effect it becomes much more likely to leave an error in the code.
